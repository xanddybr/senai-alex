Verbos HTTP — o que cada um faz

GET
Buscar dados do servidor. Não altera nada.
POST
Enviar dados e criar um novo recurso.
PUT
Atualizar um recurso existente por completo.
DELETE
Remover um recurso do servidor.
Códigos de status HTTP mais importantes
200 OK
Sucesso — dados retornados
201 Created
Recurso criado com sucesso
400 Bad Request
Dados inválidos enviados
401 Unauthorized
Não autenticado
404 Not Found
Recurso não encontrado
500 Server Error
Erro interno no servidor

Prática 1 — Fetch API: Buscando dados (GET)
15 min
API pública para praticar: https://jsonplaceholder.typicode.com — retorna dados fictícios de posts, usuários e comentários. Sem cadastro.

1. Fetch básico com .then()

fetch('https://jsonplaceholder
    .typicode.com/posts/1')
  .then(res => res.json())   // converte
  .then(data => console.log(data))
  .catch(err => console.error(err));

2. Fetch com async/await (preferido)

async function buscarPost(id) {
  const res = await fetch(
    `https://jsonplaceholder
      .typicode.com/posts/${id}`
  );
  const data = await res.json();
  return data;
}

3. Dentro de um componente React com useEffect

src/components/Posts.jsx
import { useState, useEffect }
  from 'react';

function Posts() {
  const [posts, setPosts] =
    useState([]);

  useEffect(() => {
    fetch('https://jsonplaceholder
        .typicode.com/posts')
      .then(r => r.json())
      .then(d => setPosts(d));
  }, []); // [] = executa 1 vez

  return (
    <ul>
      {posts.slice(0, 5).map(p => (
        <li key={p.id}>
          {p.title}
        </li>
      ))}
    </ul>
  );
}

export default Posts;

Prática 2 — Fetch: POST, PUT e DELETE
15 min

POST — criar um novo recurso

fetch('/api/posts', {
  method: 'POST',
  headers: {
    'Content-Type':
      'application/json',
  },
  body: JSON.stringify({
    title: 'Novo Post',
    body:  'Conteúdo...',
    userId: 1,
  }),
})
  .then(r => r.json())
  .then(data => console.log(data));

PUT — atualizar um recurso existente

fetch('/api/posts/1', {
  method: 'PUT',
  headers: {
    'Content-Type':
      'application/json',
  },
  body: JSON.stringify({
    title: 'Título atualizado',
  }),
})
  .then(r => r.json())
  .then(data => console.log(data));

DELETE — remover um recurso

fetch('/api/posts/1', {
  method: 'DELETE',
})
  .then(res =>
    console.log('Removido:',
      res.status)  // 200 ou 204
  );

Headers HTTP mais usados

// Indica o formato do corpo:
'Content-Type': 'application/json'

// Autenticação por token:
'Authorization': 'Bearer <token>'

// Formato aceito na resposta:
'Accept': 'application/json'
🧪 Teste no console: fetch('https://jsonplaceholder.typicode.com/posts', {method:'POST', headers:{...}, body: JSON.stringify({...})})

Prática 3 — Axios: instalação e comparação com Fetch
10 min

Instalação

npm install axios
Fetch API (nativo do browser)
  • Nativo do browser — sem instalar nada
  • Precisa chamar .json() na resposta
  • Não rejeita erros HTTP como 404 e 500
  • Sem timeout nativo configurável
  • Sem interceptors nativos
  • Precisa de JSON.stringify() no body
  • Precisa definir Content-Type manualmente
Axios (biblioteca externa)
  • Biblioteca pequena (~13kb gzipped)
  • Retorna data diretamente na resposta
  • Rejeita erros HTTP automaticamente
  • Timeout configurável em milissegundos
  • Interceptors prontos (ex: adicionar token)
  • Serializa o objeto JS automaticamente
  • Define Content-Type automaticamente
📦 Instale o Axios no seu projeto e substitua um fetch pelo equivalente em Axios!

Prática 4 — Axios: GET, POST e instância com baseURL
15 min

GET com Axios

import axios from 'axios';

async function buscar() {
  // data já é o objeto JS!
  // sem .json() nem JSON.stringify
  const { data } = await axios.get(
    'https://jsonplaceholder
      .typicode.com/users'
  );
  console.log(data);
}

POST com Axios — muito mais simples

async function criar(dados) {
  // objeto JS direto no 2º arg
  // axios faz JSON.stringify por você
  const { data } = await axios.post(
    '/api/usuarios',
    dados
  );
  return data;
}

Instância com axios.create() — padrão recomendado para projetos reais

src/services/api.js
import axios from 'axios';

// Configura uma instância centralizada
const api = axios.create({
  baseURL: 'https://jsonplaceholder
    .typicode.com',
  timeout: 5000,
  headers: {
    'Content-Type': 'application/json',
  },
});

export default api;
Uso em qualquer componente
import api from '../services/api';

// baseURL já está configurada
// só precisa do caminho relativo
const { data } = await api.get(
  '/posts'
);

const { data: novo } = await api.post(
  '/posts', { title: 'Novo' }
);
🗂️ Crie o arquivo src/services/api.js com sua instância Axios e use-a para buscar /users!

Prática 5 — Loading State e Tratamento de Erros
15 min

Componente completo com os três estados: loading, erro e dados

src/components/Usuarios.jsx
import { useState, useEffect }
  from 'react';
import api from '../services/api';

function Usuarios() {
  const [users,   setUsers]   =
    useState([]);
  const [loading, setLoading] =
    useState(true);
  const [error,   setError]   =
    useState(null);

  useEffect(() => {
    api.get('/users')
      .then(({ data }) =>
        setUsers(data))
      .catch(err =>
        setError(err.message))
      .finally(() =>
        setLoading(false));
  }, []);

  if (loading)
    return <p>Carregando...</p>;

  if (error)
    return <p>Erro: {error}</p>;

  return (
    <ul>
      {users.map(u => (
        <li key={u.id}>
          <strong>{u.name}</strong>
          <br />
          {u.email}
        </li>
      ))}
    </ul>
  );
}

export default Usuarios;
1. Sempre usar .catch()
Erros de rede ou HTTP devem ser capturados e exibidos ao usuário de forma amigável — nunca deixe a requisição silenciosa.
2. Estado de loading
Sempre exiba um feedback visual enquanto aguarda a resposta. O usuário precisa saber que algo está acontecendo.
.finally()
Executa sempre — tanto em sucesso quanto em erro. Ideal para resetar o estado de loading.
⚠️ Cuidado com Fetch!
O Fetch não lança erro para status 4xx/5xx. Sempre verifique if (!res.ok) throw new Error(...) antes do .json().

Desafio de Integração — Aula 03
20 min

Construa um buscador de usuários consumindo uma API real

1
Listar usuários
Busque /users da JSONPlaceholder e exiba uma lista com nome, e-mail e cidade de cada usuário.
2
Detalhe — posts do usuário
Ao clicar em um usuário, faça uma segunda requisição para /posts?userId=X e exiba os posts dele na mesma tela.
3
Estado de loading
Mostre "Carregando..." enquanto cada requisição está em andamento. Use um estado loading separado por chamada.
4
Tratamento de erro
Simule um erro trocando a URL por uma inválida e exiba uma mensagem amigável com um botão para tentar novamente.
5
Instância Axios
Use axios.create() com baseURL em src/services/api.js para todas as chamadas — sem repetir a URL base.
