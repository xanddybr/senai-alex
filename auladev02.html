SPA e Roteamento no Front-End
MPA — Multi-Page App
  • Cada página = nova requisição ao servidor
  • O browser recarrega completamente a cada clique
  • Exemplo: sites em PHP, HTML estático
SPA — Single-Page App
  • Uma única página HTML é carregada
  • O conteúdo muda via JavaScript, sem reload
  • Exemplo: React, Vue, Angular
React Router v6 — biblioteca oficial de roteamento

Permite criar rotas declarativas dentro do componente React, sem recarregar a página. O React Router "engana" o browser, trocando componentes enquanto a URL muda.

BrowserRouter Routes Route Link useNavigate useParams Outlet

Prática 1 — Instalando o React Router v6
10 min
1
Instalar o pacote
npm install react-router-dom
2
Envolver o app com BrowserRouter em main.jsx
src/main.jsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import { BrowserRouter } from 'react-router-dom' // 1. Importa o router
import App from './App.jsx'
import './index.css'

createRoot(
  document.getElementById('root')
).render(
  <StrictMode>
    <BrowserRouter>          {/* 2. Envolve o App */}
      <App />
    </BrowserRouter>
  </StrictMode>,
);
3
Criar a estrutura de páginas
Estrutura de pastas recomendada
src/
  pages/
    Home.jsx       ← página principal
    Sobre.jsx      ← página "Sobre nós"
    Contato.jsx    ← página de contato
  App.jsx          ← define as rotas
  main.jsx         ← ponto de entrada
💡 Crie as três páginas como componentes vazios antes de continuar — só com um <h1> no return!

Prática 2 — Criando Rotas Básicas
15 min

App.jsx — definindo as rotas da aplicação

src/App.jsx
import { Routes, Route, Link }
  from 'react-router-dom';
import Home    from './pages/Home';
import Sobre   from './pages/Sobre';
import Contato from './pages/Contato';

function App() {
  return (
    <div>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/sobre">Sobre</Link>
        <Link to="/contato">Contato</Link>
      </nav>

      <Routes>
        <Route path="/"
          element={<Home />} />
        <Route path="/sobre"
          element={<Sobre />} />
        <Route path="/contato"
          element={<Contato />} />
      </Routes>
    </div>
  );
}

export default App;
<Routes>
Container que avalia qual <Route> ativar com base na URL atual.
<Route path>
Define o caminho de URL que ativa o componente. path="/" é a raiz.
element={}
Recebe o componente JSX que será renderizado quando a rota casar.
<Link to>
Substitui o <a href>. Navega sem recarregar a página — essencial na SPA.
🚀 Teste cada rota no browser: /, /sobre e /contato — observe que não há reload!

Prática 3 — Rotas Dinâmicas e useParams
15 min
Caso de uso: ao clicar em um produto de uma lista, a URL muda para /produto/42 e a página de detalhe carrega os dados daquele produto específico.

1. Definindo a rota com parâmetro :id

src/App.jsx
<Route
  path="/produto/:id"
  element={<Produto />}
/>

2. Criando links com id dinâmico

src/pages/Produtos.jsx
const produtos = [
  { id: 1, nome: 'Teclado' },
  { id: 2, nome: 'Mouse' },
  { id: 3, nome: 'Monitor' },
];

{produtos.map(p => (
  <Link to={`/produto/${p.id}`}>
    {p.nome}
  </Link>
))}

3. Lendo o parâmetro com useParams

src/pages/Produto.jsx
import { useParams }
  from 'react-router-dom';

function Produto() {
  // id vem da URL: /produto/42
  const { id } = useParams();

  return (
    <div>
      <h1>Produto #{id}</h1>
      <p>
        Carregando dados
        do produto {id}...
      </p>
    </div>
  );
}

export default Produto;
🎯 Crie links para /produto/1, /produto/2, /produto/3 e exiba o id em cada página!

Prática 4 — Rota 404 e Rotas Aninhadas
15 min

Capturando URLs inexistentes com path="*"

src/App.jsx — adicionar dentro de <Routes>
<Routes>
  <Route path="/" element={<Home />} />
  <Route path="/sobre" element={<Sobre />} />
  <!-- SEMPRE por último: captura tudo -->
  <Route
    path="*"
    element={<PaginaNaoEncontrada />}
  />
</Routes>

O asterisco * casa com qualquer URL que não bateu nas rotas anteriores. É a "rota coringa" — deve ser sempre a última dentro do <Routes>.

Rotas aninhadas com <Outlet />

src/App.jsx — rota pai com filhas
<Route
  path="/painel"
  element={<Painel />}
>
  <Route
    path="perfil"
    element={<Perfil />}
  />
  <Route
    path="config"
    element={<Config />}
  />
</Route>
src/pages/Painel.jsx — componente pai
import { Outlet, Link }
  from 'react-router-dom';

function Painel() {
  return (
    <div>
      <h2>Painel</h2>
      <nav>
        <Link to="perfil">
          Meu Perfil
        </Link>
        <Link to="config">
          Configurações
        </Link>
      </nav>

      {/* filho renderiza aqui */}
      <Outlet />
    </div>
  );
}
⚙️ Adicione a rota path="*" ao seu projeto e teste digitar uma URL que não existe!

Prática 5 — Navegação Programática com useNavigate
15 min
Quando usar: redirecionar após o submit de um formulário, após login bem-sucedido, ou ao clicar em um botão — casos em que <Link> não é suficiente.
src/pages/FormLogin.jsx
import { useNavigate }
  from 'react-router-dom';

function FormLogin() {
  const navigate = useNavigate();

  function handleSubmit(e) {
    e.preventDefault();

    // ... lógica de autenticação ...

    // redireciona para o painel
    navigate('/painel');

    // navegar para trás (voltar)
    // navigate(-1);

    // substituir entrada no histórico
    // navigate('/home', { replace: true });
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        placeholder="Usuário"
      />
      <input
        type="password"
        placeholder="Senha"
      />
      <button type="submit">
        Entrar
      </button>
    </form>
  );
}

export default FormLogin;

Opções do navigate()

navigate('/rota')
Empurra uma nova entrada no histórico do browser. O usuário pode voltar com o botão ←.
navigate(-1)
Volta uma página — equivale ao botão Voltar do browser.
navigate(1)
Avança uma página no histórico.
{ replace: true }
Substitui a entrada atual do histórico. O usuário não consegue voltar para a rota anterior. Útil após logout.
🧪 Desafio: crie um formulário de login que redireciona para /painel ao submeter!

Desafio de Integração — Aula 02
20 min

Construa um mini-site de produtos com React Router

1
Estrutura de páginas
Crie 4 páginas: Home, Produtos, Produto/:id e NaoEncontrada. Configure todas no App.jsx com <Routes> e <Route>.
2
Navegação
Barra de navegação com <Link> para Home e Produtos. Estilize o link da rota ativa usando NavLink e a prop className.
3
Lista de produtos
Página Produtos: renderize um array de 5 produtos usando .map(). Cada item deve ter um <Link to={`/produto/${p.id}`}>.
4
Detalhe do produto
Página Produto/:id: use useParams para ler o id da URL. Exiba o nome do produto filtrando o array pelo id recebido.
5
Página 404
Rota path="*" com uma página personalizada. Deve ter um botão "Voltar para Home" que usa useNavigate() para redirecionar.
🌟 Bônus: use useState para guardar o array de produtos no App.jsx e passe-o como prop para as páginas — ou armazene em um arquivo data/produtos.js e importe onde precisar.
